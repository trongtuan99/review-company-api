## Common Scenarios

### Scenario 1: New Developer Starting Monday
1. Sunday evening: Read `README.md` (skim it, 5 min)
2. Monday morning: Full read of `README.md` (10 min)
3. Monday afternoon: Setup and `docs/codebase-summary.md` (20 min)
4. Tuesday: `docs/code-standards.md` (30 min)
5. Wednesday: Reference as needed

1. Check `docs/project-overview-pdr.md` for requirements
2. Reference `docs/code-standards.md` for naming and patterns
3. Check `docs/codebase-summary.md` for similar components
4. Check `docs/system-architecture.md` for data flow

1. Check `docs/codebase-summary.md` to find the component/service
2. Reference `docs/system-architecture.md` to understand data flow

1. Check `docs/project-overview-pdr.md` for performance targets
2. Review `docs/system-architecture.md` for performance architecture
3. Check `docs/code-standards.md` for optimization patterns
4. Review `docs/codebase-summary.md` for optimization details

1. Reference `docs/code-standards.md` for standards
2. Check `docs/system-architecture.md` for design patterns

2. `docs/codebase-summary.md` (Architecture Overview + Directory Structure) - 15 min
3. `docs/code-standards.md` (General Principles + Naming) - 5 min

### Intermediate Understanding (1-2 hours)
- Complete the Basic path
- `docs/code-standards.md` (Component Structure + Hooks) - 20 min
- `docs/system-architecture.md` (High-Level + Component Architecture) - 30 min

### Advanced Understanding (2-4 hours)
- Complete the Intermediate path
- Full read of `docs/system-architecture.md` - 45 min

### Expert Understanding (4+ hours)
- Complete all above paths
- Full read of all documentation
- Deep dive into specific areas as needed
